# FS Memory System

Progressive, self-compressing memory backed by local files. Every session is journaled. At the end of each session you MUST update memory.

---

## File Layout

```
memory/
├── AGENT.md              ← Active context injected every session (auto-rebuilt, bounded ~2KB)
├── profile.md            ← Persistent user/project identity and preferences (human-editable)
├── journals/
│   ├── YYYY-MM.journal.md      ← Raw session entries appended by mem journal
│   └── YYYY-MM.condensed.md   ← LLM-summarized version (auto-generated by mem condense)
├── topics/
│   └── promoted.md       ← Long-term facts auto-promoted from sessions
└── archive/
    └── YYYY-MM.journal.md ← Entries older than 90 days, compressed to 1 line each
```

---

## Commands

```bash
mem read <path>                          # Read a file from memory/ by relative path
mem write <path> <content>              # Overwrite a memory file
mem append <path> <content>             # Append to a memory file
mem list [subdir]                        # List files in a subdirectory
mem journal <projectDir> <ctx> <summary> # Append a session entry (MANDATORY)
mem condense                             # Run the full condensation pipeline
mem status                               # Show AGENT.md size + preview
```

## Session Protocol (MANDATORY)

### During the session — write at milestones:
Append a journal entry whenever:
- A bug is fixed or a feature is shipped
- An architectural decision is made
- The user says something like "always", "never", "from now on"
- You hit a surprising or non-obvious insight

```bash
mem journal "eval/src/memory" "condensation pipeline" \
  "Rewrote condense.ts to use opencode run instead of direct Gemini SDK.
  DECIDED: project never calls model APIs directly — always routes through opencode."
```

### Permanent facts — flag with special prefixes:
Prefix lines to trigger auto-promotion to `topics/promoted.md`:
- `IMPORTANT:` — must-remember fact
- `DECIDED:` — architectural/design decision
- `PERMANENT:` — user preference that never changes
- `KEY FACT:` — critical project constraint
- `ALWAYS:` / `NEVER:` — hard rules

### End of session — REQUIRED steps:
When the user signals end ("done", "good night", "wrap up", "I'm off"):

1. **Append journal** — run `mem journal` with the full session summary
2. **Update profile** — if persistent facts emerged, write to profile.md:
   ```bash
   mem write profile.md "# Profile\n\n- Uses bun, not npm\n- Project: adival eval framework"
   ```
3. **Run condensation** (optional — keeps AGENT.md fresh):
   ```bash
   mem condense
   ```

## Reading Memory

**At session start**, `memory/AGENT.md` is auto-injected into context (bounded ~2KB).

To read specific past entries directly:
```bash
mem read profile.md
mem read journals/2026-03.journal.md
mem read topics/promoted.md
mem list journals
```

---

## Condensation Pipeline

```bash
mem condense   # summarize yesterday, promote facts, compress old entries, rebuild AGENT.md
mem status     # check AGENT.md current size and preview
```

**What condense does:**
1. LLM-summarizes yesterday's raw journal → `journals/YYYY-MM.condensed.md`
2. Promotes permanent facts → `topics/promoted.md`
3. Age-compresses all journal months in-place:
   - 0–6 days: unchanged
   - 7–29 days: 2-3 sentences
   - 30–89 days: 1 sentence
   - 90+ days: archived to `memory/archive/`
4. Rebuilds `AGENT.md` with a hard 2KB cap

---

## Promotion Algorithm

A session note becomes permanent in `topics/promoted.md` if:
- Contains an explicit flag prefix (always promoted)
- Matches existing memory on 50%+ of keywords (recurrence boost)
- LLM judge confirms it's a durable architectural/preference fact
